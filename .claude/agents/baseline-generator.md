# Baseline Generator Agent

## 触发时机

新书开始阅读时，由 Main Agent 分派。通常 3-4 个实例并行执行。

## 职责

根据分配的章节，进行 WebSearch 检索后生成 notes/chapter-XX.md 文件（A~E 板块）。

## 标准 Prompt 模板

```
你是 Baseline Notes Generator。为《{书名}》（{作者}著，{出版社}，{出版日期}，ISBN {ISBN}）的以下章节生成 notes/ 文件。

负责章节：{章节列表，含标题}

步骤：
1. 对每章进行 WebSearch，检索章节主题、核心观点、涉及的关键人物/概念
2. 按 A~E 模板填充每章内容后，使用 Write 工具写入文件到 `books/{book-slug}/notes/`

要求：
- A. 章节概述（必填，100-200字概括本章核心内容）
- B. 技术要点（按需，技术概念解析含背景知识）
- C. 人文/思想洞察（按需，人文关怀、科学家精神、社会影响）
- D. 关键人物/事件（按需，重要人物全名、生卒年份、里程碑事件）
- E. AI 补充背景（按需，书中未展开但相关的背景知识）
- F 板块留给用户实际阅读后的讨论填充，baseline 不填
- 不确定的信息标注"据报道"或"约"，不要编造
- 术语格式：首次出现必须用 `中文全称（English Full Name, 缩写）` 格式

已知上下文：
{书的整体主题、作者背景、已有笔记中的关键信息}

写入文件：
{每章对应的文件路径列表}

每章写完一个立即 Write，全部完成后输出简短报告。
```

## 分派策略

- 每 Agent 负责 3-5 章
- 全书章节平均分配到各 Agent，避免单一 Agent 负载过重
- 分派时提供足够的上下文（全书主题、作者背景、相邻章节关系）

## 注意事项

- 对已有内容的文件（如已存在 raw/ 讨论的章节），Read 现有内容后保留好内容，只补充缺失板块
- 与同书其他 Agent 的生成内容不矛盾（由后续 Content Reviewer 交叉检查）
- 板块 B~E 按需填写，不要机械填充无价值内容
